Si desidera realizzare un'applicazione grafica che utilizzi le componenti swing (javax.swing) e che consenta:

  • di inserire un valore di temperatura espresso in gradi Celsius in un apposito campo di testo;
  • di convertire la temperatura in gradi Celsius nella corrispondente temperatura in gradi Fahrenheit mediante la pressione di un pulsante;
  • di visualizzare il valore della temperatura in gradi Fahrenheit in un'etichetta della GUI (vedi Fig. 1).

Si richiede di implementare la GUI mediante un'unica classe Java di nome CelsiusConverterGUI contenente il metodo main() necessario per avviare l'applicazione grafica.

Svolgimento Esercizio 1

Proviamo a progettare la GUI di CelsiusConverterGUI procedendo per gradi, separando la definizione del layout dalla gestione dell'interazione con l'utente. In particolare, nella Fase 1 svilupperemo una GUI che presenta il layout desiderato ma che non è interattiva (non risponde agli input dell'utente) e nella Fase 2 provvederemo a rendere la GUI interattiva secondo quanto specificato nel testo dell'esercizio.

Fase 1: GUI non interattiva

Definire il layout della GUI significa specificare in modo appropriato:

  1. le componenti grafiche (componenti swing) utilizzate e la relativa gerarchia d'inclusione (alcune componenti swing, componenti di tipo Container, fungono da contenitori e possono contenere a loro volta altre componenti Container che includono ulteriori gruppi di componenti e così via);
  2. l'organizzazione spaziale delle componenti, più specificatamente per ogni componente di tipo Container è necessario specificare la disposizione spaziale delle componenti che contiene.
Fase 1.1: Gerarchia d'inclusione delle componenti

Componenti swing della GUI:

  • javax.swing.JFrame: finestra principale della GUI e root della gerarchia di inclusione;
  • java.awt.Container: content pane del JFrame;
  • javax.swing.JTextField: campo per l'inserimento del testo;
  • javax.swing.JButton: pulsante della GUI;
  • javax.swing.JLabel: etichetta a destra del campo di testo;
  • javax.swing.JLabel: etichetta a destra del pulsante.

Fase 1.2: Gestione del layout delle componenti di tipo Container

La gerarchia d'inclusione delle componenti prevede la presenza di una sola componente di tipo Container: il content pane del JFrame. Come gestore di layout si utilizzerà il javax.swing.GroupLayout, la Fig. 3 illustra lo pseudo-codice per il layout orizzontale e verticale delle componenti, tale pseudo-codice consente di ottenere il layout globale desiderato. Si è adottato lo stesso pseudo-codice introdotto nel tutorial sulle swing (per i dettagli consultare il tutorial Java How to Use GroupLayout).

GroupLayout Pseudo Code

horizontal layout = sequential group {parallel group (LEFT){c1, c3}, parallel group (LEFT){c2, c4}}

vertical layout = sequential group {parallel group (BASELINE){c1, c2}, parallel group (BASELINE){c3, c4}}

Fase 2: GUI interattiva

Per rendere una GUI interattiva è necessario collegare le componenti swing con le quali interagisce l'utente, tramite il mouse e/o la tastiera, ad oggetti ascoltatori di eventi (event listener object) che recepiscono l'input dell'utente e definiscono il conseguente comportamento del programma. Nel nostro caso è necessario associare un oggetto ascoltatore al JButton specificando che ad ogni click dell'utente il testo del componente JLabel deve essere aggiornato. A tale scopo la classe JButton prevede il metodo

public void addActionListener(ActionListener l)

che prende come parametro un oggetto di tipo ActionListener, cioè un oggetto che implementa l'interfaccia java.awt.event.ActionListener. Una classe implementa l'interfaccia ActionListener se implementa il seguente metodo

   public void actionPerformed(ActionEvent e)

in cui è possibile specificare il codice da eseguire a seguito dell'input dell'utente; tale metodo riceve in input un oggetto di tipo java.awt.event.ActionEvent che viene generato automaticamente ogni volta che viene registrato un input dell'utente (click sul pulsante) e che contiene tra l'altro informazioni inerenti al tipo di evento e alla componente che l'ha generato. Molto spesso l'oggetto di tipo ActionListener è lo stesso che rappresenta il JFrame; in questo esercizio adotteremo tale approccio.

NetBeans IDE: aspetti fondamentali

Non è necessario conoscere ogni dettaglio dell'IDE NetBeans per descrivere come è possibile utilizzarlo nello sviluppo di applicazioni utente grafiche basate sul framework Swing. Infatti, limitatamente allo sviluppo di una GUI è sufficiente conoscere solo il componente Palette, l'area di disegno (Design Area), il Property Editor, e l'Inspector (o Navigator). Tali caratteristiche saranno discusse nel seguito.

Il componente Palette

Si tratta di un contenitore (tavolozza) di tutte le componenti fornite dalle Swing API. La funzione di gran parte di tali componenti è facilmente intuibile. Nella nostra applicazione utilizzeremo soltanto JLabel, JTextField e JButton.

Il componente Design Area

Il componente Design Area, area di disegno, è la parte dell'IDE in cui verrà costruita graficamente la GUI. Presenta due viste: source view, e design view. Design view è il default; è possibile passare da una vista all'altra, in qualsiasi momento, cliccando nelle rispettive schede (tab). Inizialmente viene visualizzato un singolo oggetto JFrame rappresentato da un rettangolo con sfondo grigio e contorni blue. Il comportamento tipico (come l'uscita del programma quando l'utente chiude la finestra) è generato automaticamente dall'IDE. Selezionando la vista source view, viene visualizzato il codice sorgente della classe incluso il codice generato automaticamente. Il codice generato in modo automatico è contenuto all'interno di sezioni non editabili contraddistinte da uno sfondo di colore diverso. In particolare, si può osservare che l'IDE crea il metodo privato initComponents(), dedicato all'istanziazione e all'inizializzazione delle componenti grafiche della GUI. Tale metodo specifica l'opzione EXIT_ON_CLOSE, effettua alcune operazioni per la gestione del layout e impacchetta le componenti invocando il metodo pack().

Il Property Editor

Come è deducibile dal nome, il Property Editor permette di editare le proprietà di ogni componente in modo semplice ed intuitivo. In pratica, una volta selezionato il componente di interesse, tale editor mostra l'elenco delle proprietà di quel componente unitamente a dei controlli che permettono di modificarle in modo agevole, evitando di lavorare direttamente sul codice sorgente.

L'Inspector o (Navigator)

L'ultimo componente del GUI builder di Netbeans è l'Inspector o (Navigator). L'Inspector fornisce una rappresentazione grafica delle componenti dell'applicazione. Si rivela particolarmente utile per visualizzare e navigare la gerarchia di inclusione delle componenti della GUI e, in particolare, per assegnare dei nomi significativi alle varie componenti Swing in sostituzione di quelli di default generati in modo automatico dal GUI builder.

Implementazione di CelsiusConverterGUI mediante il GUI builder di Netbeans

Nel seguito illustreremo come usare l'IDE NetBeans per creare la GUI dell'applicazione CelsiusConverterGUI. Trascinando ogni componente dalla Palette all'area di disegno, l'IDE genererà in modo automatico il codice sorgente appropriato.

Step 1: Impostazione del Titolo

Per prima cosa, impostare il titolo del JFrame dell'applicazione come Celsius Converter, cliccando sul JFrame nell'Inspector e specificando il titolo con il Property Editor. Ciò è fattibile o mediante un doppio click sulla proprietà title e digitando il titolo direttamente, o cliccando sul pulsante con icona "..." e digitando il titolo nel form che compare.

Step 2: Aggiunta del JTextField

Trascinare il componente JTextField dalla Palette all'angolo in alto a sinistra dell'area di disegno. Non appena il componente si trova nelle vicinanze dell'angolo in alto a sinistra, l'IDE visualizzerà delle linee tratteggiate (visual cues) che suggeriscono la spaziatura appropriata. Usando queste linee come guida, rilasciare (drop) il JTextField nella posizione desiderata. Sebbene il nome di default jTextField1 potrebbe essere ritenuto poco significativo, evitare per il moneto di modificarlo. In seguito si vedrà come è possibile rimpiazzare i nomi di default ed in generale come impostare le proprietà di ogni componente in un secondo momento.

Step 3: Aggiunta del primo JLabel

Trascinare il componente JLabel nell'area di disegno. Piazzarlo alla destra del JTextField, utilizzare i visual cues per trovare la spaziatura ottimale. Assicurarsi anche che la base-line del testo per questo componente sia allineata a quella del JTextField. I visual cues forniti dall'IDE dovrebbero semplificare tale operazione.

Step 4: Aggiunta del JButton

Trascinare il componente JButton dalla Palette e posizionarla in prossimità del margine sinistro dell'area di disegno, esattamente sotto il JTextField. Di nuovo, aiutarsi con i visual cues per trovare un allineamento e una spaziatura ottimale. Sebbene la larghezza del JButton e del JTextField potrebbe essere ritenuta non ottimale, evitare per il moneto di modificarla. Nel seguito si vedrà come è possibile impostare le proprietà di ogni componente in modo da produrre il layout desiderato.

Step 5: Aggiunta del secondo JLabel

In fine, aggiungere il secondo JLabel, ripetendo la procedura vista allo Step 2. Piazzare questa seconda etichetta alla destra del JButton.

Aggiustamenti finali della GUI

Una volta piazzati i componenti della GUI nell'area di disegno, è necessario effettuare alcuni piccoli aggiustamenti finali. Ciò è fattibile in molti modi diversi, l'approccio che verrà illustrato è soltanto una tra i possibili.

Step 6: Impostazione del testo dei componenti

Prima, effettuare doppio click nel JTextField e nel JButton per cambiare il testo di default inserito dall'IDE. Una volta cancellato il testo del JTextField, la dimensione del componente si restringerà. Cambiare il testo del JButton da jButton1 a Convert. Inoltre, cambiare il testo dell'etichetta in alto in Celsius e di quella in basso in Fahrenheit.

Step 7: Impostazione della dimensione dei componenti

Tenendo premuto il tasto Shift, cliccare sul JTextField e poi sul JButton. Tali componenti saranno evidenziati nella visualizzazione per mostrare che sono entrambi selezionati. Cliccare con il tasto destro del mouse sull'area evidenziata, e selezionare Same Size → Same Width. Ai componenti sarà assegnata la stessa larghezza. Assicurarsi che tra i componenti selezionati non vi sia il JFrame stesso, in caso contrario la voce Same Size del menù sarà disattivata.

Step 8: Rimozione dello spazio inutilizzato

In fine, afferrare la maniglia nell'angolo in basso a destra del JFrame e aggiustare la sua dimensione eliminando lo spazio inutilizzato. Nota che l'eliminazione di tutto lo spazio non occupato può comportare una visualizzazione parziale del titolo. L'utente finale dell'applicazione può ridimensionare la GUI a piacimento, ma è importante lasciare uno spazio extra sul lato destro per assicurarsi che ogni componente (in particolare ogni etichetta) sia correttamente visualizzata.

Inserimento della logica dell'applicazione

Una volta costruito il layout della GUI, è necessario implementare la logica dell'applicazione.

Step 9: Cambiare i nomi di default delle variabili

Nell'Inspector vengono visualizzati i nomi di default assegnati dall'IDE alle componenti Swing inseriti nel JFrame. Per ogni componente, viene prima mostrato il nome della variabile e poi, tra parentesi quadre, il nome del corrispondente oggetto. Ad esempio, jtextField1 [JTextField] significa che jTextField1 è il nome della variabile e JTextField il suo tipo. I nomi di default non sono molti rilevanti nel contesto di questa applicazione, pertanto è consigliabile cambiarli con nomi più significativi. Right-click su ogni nome di variabile e scegliere Change variable name. Modificare i nomi di variabile nel seguente modo:

  • jTextField1 → tempTextField
  • jLabel1 → celsiusLabel
  • jButton1 → convertButton
  • jLabel2 → fahrenheitLabel

Ogni cambiamento effettuato nell'Inspector corrisponderà ad un aggiornamento automatico del codice sorgente. Assicurarsi che la compilazione non fallisca a causa di possibili errori di digitazione o errori che possono essere commessi quando si edita manualmente il codice.

Step 10: Inserimento (registrazione) degli ascoltatori di eventi

Quando un utente finale interagisce con un componente Swing di una GUI (ad esempio cliccando sul pulsante Convert), quel componente genererà un tipo di oggetto speciale—detto oggetto evento o event object—che sarà trasmesso a tutti gli altri oggetti che sono stati precedentemente registrati come ascoltatori (listener) per quell'evento. L'IDE NetBeans rende la registrazione degli eventi estremamente semplice:

Nell'area di disegno, cliccare sul pulsante Convert per selezionarlo. Assicurarsi che solo questo pulsante sia selezionato (se anche il JFrame è selezionato, questo step non funzionerà). Right-click sul pulsante Convert e scegliere Events → Action → ActionPerformed. Ciò genererà lo scheletro (metodi con corpi vuoti) del codice necessario per la gestione dell'evento, lasciando al programmatore l'onere di inserire le istruzioni relative al suo specifico problema.

Step 11: Inserimento del codice per la conversione della temperatura

Lo step finale consiste nell'inserimento del codice per la conversione della temperatura nel corpo del metodo convertButtonActionPerformed(). Il codice seguente è quanto necessario inserire per ottenere la temperatura in gradi Fahrenheit da quella in gradi Celsisus:

    //Parse degrees Celsius as a double and convert to Fahrenheit.
    int tempFahr = (int)((Double.parseDouble(tempTextField.getText())) * 1.8 + 32);
    fahrenheitLabel.setText(tempFahr + " Fahrenheit");

Una volta inserito tale codice nel corpo del metodo convertButtonActionPerformed() l'implementazione della classe CelsiusConverterGUI è conclusa.

Step 12: Esecuzione dell'applicazione

Eseguire l'applicazione consiste semplicemente nel selezionare Run → Run Main Project dell'IDE NetBeans. La prima volta che viene eseguita questa applicazione, verrà richiesto in una finestra di dialogo di impostare CelsiusConverterGUI come la classe main per questo progetto. Scegliere OK, e quando il programma termina la compilazione, sarà possibile vedere l'applicazione in esecuzione nella propria finestra.
